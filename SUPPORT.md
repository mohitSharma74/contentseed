# Support

Support is provided on a best-effort basis.

## Where to Ask

- GitHub Discussions: questions, ideas, and general usage help
- GitHub Issues: reproducible bugs and actionable feature requests

## Before Opening an Issue

1. Check existing issues and discussions.
2. Confirm the issue reproduces on the latest `main`.
3. Include clear reproduction steps and environment details.

## Security Issues

For vulnerabilities, use the private channel in [SECURITY.md](./SECURITY.md).
