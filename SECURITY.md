# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Report vulnerabilities privately to:

- mohit.sharma24546@gmail.com

Include the following details when possible:

- affected version or commit SHA
- reproduction steps
- impact assessment
- suggested remediation (if known)

## Response Expectations

Support is provided on a best-effort basis.

- initial acknowledgement target: within 7 days
- status updates: provided when new information is available

## Disclosure Process

1. We acknowledge receipt and validate the report.
2. We investigate and prepare a fix.
3. We coordinate disclosure timing with the reporter.
4. We publish a fix and release notes.

## Scope Notes

ContentSeed runs client-side and can call third-party LLM APIs directly from the browser.
Do not include active API keys in reports.
